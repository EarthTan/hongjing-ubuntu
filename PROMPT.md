# 任务:红警风格 2D RTS(Ubuntu 原生)—— Ralph Loop 循环体

你运行在一个自动循环中:本提示词会被反复执行,每次执行你都是全新的、无记忆的。
**所有状态只存在于文件和 git 里。你的记忆就是 repo。**

---

## 第 0 步:定位(每轮必做,禁止跳过)

按顺序执行:

1. 若 repo 不存在 → `git init`,创建 PROGRESS.md(下方模板)、DECISIONS.md、ISSUES.md,首次 commit,然后**直接进入第 1 步做 MVP-1**
2. 若 repo 已存在 → 依次读:
   - `PROGRESS.md`(唯一真相源)
   - `ISSUES.md`(历史踩坑,禁止重复已失败的方案)
   - `git log --oneline -10`
   - 只读与本轮目标项相关的源码文件,**禁止一次性读全 repo**
3. 找到 PROGRESS.md 中第一个 `[ ]` 项 → 这就是本轮唯一目标
4. 若 10 项全部 `[x]` → 跳到「终局验收」

## 第 1 步:实现(本轮只做这一项)

**每轮循环只完成一个 MVP 项。做完就收尾退出,禁止顺手开始下一项。**
宁可一项拆成两轮做完(在 PROGRESS.md 该项下追加子任务 checkbox),也禁止一轮做两项。

## 第 2 步:自测(headless,必须真实运行)

- 所有自动化测试用 `SDL_VIDEODRIVER=dummy SDL_AUDIODRIVER=dummy python -m pytest tests/` 运行
- 每个 MVP 项至少写 1 个对应的 headless 测试(逻辑层测试:寻路结果、资源数值、建造队列状态等,不测像素)
- 另跑冒烟测试:`tests/smoke_test.py` 以 dummy driver 启动游戏主循环空跑 300 帧不 crash,每轮都要跑(回归)
- 测试不过 → 修,禁止把失败项打勾

## 第 3 步:落盘收尾(每轮必做)

1. PROGRESS.md 对应项打勾 `[x]`,附一行完成摘要
2. 本轮所有决策追加到 DECISIONS.md:`- [MVP-N] 决策内容 —— 一句话理由`
3. `git add -A && git commit -m "feat: [N/10] 项目名"`
4. `git push`,失败处理:重试 3 次(间隔 30s)→ 切换 SSH/HTTPS 再试 → 仍失败则在 ISSUES.md 记录后本轮结束(下轮 commit 会带上补 push);**禁止因网络问题终止或阻塞**
5. 收尾后立即结束本轮,不要开始新工作

## 卡死熔断(防止循环空转)

- 本轮如果尝试修同一个错误 3 次仍失败:在 ISSUES.md 记录 `- [MVP-N][BLOCKED] 现象 / 已尝试方案 / 下一步建议换的方案`,commit 后结束本轮
- 下一轮读到 BLOCKED 记录时,**必须换一个与已尝试方案不同的实现路线**(简化功能也算换路线),禁止原样重试
- 同一项累计 BLOCKED 3 轮 → 允许降级实现(在 DECISIONS.md 写明降级内容),打勾放行,禁止无限卡在一项上

## 技术栈(钉死,禁止更换)

- Python 3.12 + Pygame-CE,单 repo,模块化:`engine/ units/ ui/ ai/ tests/`
- 美术全部程序化几何图形/色块生成,禁止外部素材文件
- 依赖写入 `requirements.txt`

## MVP 范围(10 项,顺序即依赖顺序,按序完成)

1. 可缩放窗口 + 瓦片地图(草地/水/矿)+ 边缘滚动 + 小地图
2. 建造系统:建造厂/电厂/矿场/兵营/战车工厂,电力 + 建造队列
3. 资源循环:矿车自动采矿 → 回矿场 → 金钱增加
4. 单位:步兵/火箭兵/轻坦/重坦,独立属性(HP/攻击/射程/速度)
5. 操作:框选、右键移动/攻击、A 键攻击移动、Ctrl+数字编队
6. A* 寻路 + 避障
7. 战斗表现:血条、命中闪烁、爆炸粒子、死亡移除
8. 敌方 AI:自动建造采矿,每 2 分钟一波进攻
9. 胜负:摧毁对方建造厂,显示胜利/失败画面
10. 主菜单(开始/退出)

## 决策规则

- 全自主,禁止提问、禁止等待确认
- 所有未明确事项采用最简单可行方案,每个决策写入 DECISIONS.md
- 优先保证 headless 可测的逻辑层正确,视觉表现从简

## 终局验收(全部 10 项打勾后执行,通过前禁止宣布完成)

1. `python main.py` 一条命令启动成功
2. 写并运行 `tests/full_game_test.py`:headless 模式下用脚本驱动完整一局(建基地 → 采矿 → 造兵 → 摧毁 AI 建造厂 → 触发胜利画面状态),全程无 crash
3. PROGRESS.md 10/10、DECISIONS.md 记录完整
4. `git push` 成功,远程与本地 HEAD 一致(`git status` 确认)
5. 全部通过 → 创建 `DONE` 空文件并 commit push,然后输出一行:

```
RALPH_TASK_COMPLETE
```

若第 0 步发现 `DONE` 文件已存在 → 直接输出 `RALPH_TASK_COMPLETE` 并结束,什么都不要做。
